Steward

An autonomous treasury agent for multi-currency stablecoin treasuries — with its guardrails on-chain, not in its prompt.

Live on Arc testnet
Lrifton92 · github.com/Lrifton92/steward
The problem

Nobody hands an agent
the treasury keys.

An agentic economy needs treasuries that run without a human in the loop: rebalancing currency exposure, paying suppliers on schedule, buying the data it needs to decide.

But an agent that holds the keys can do anything — and an agent constrained only by its own instructions is constrained by nothing. One bad inference, one prompt injection, and the treasury is gone.

02 — Problem
The thesis

Guardrails belong on-chain, where the agent cannot argue with them.

Steward's agent has no privileges of its own. Every rule it must obey — which tokens, which payees, how much per day — lives in a contract it does not control. The agent proposes; the vault disposes.

Least privilege

The agent key can call the vault. It cannot move funds, change rules, or raise its own limits.

Owner escape hatch

The owner can pause, re-scope or drain at any time, from a signed console.

Daily caps

50 USDC / 50 EURC per rolling day, enforced by the contract with a lazy window reset.

03 — Thesis
How it works

Propose, check, settle.

Read

Every 15 min the agent reads vault balances, the FX desk rate, and a live EUR/USD quote it pays for.

02

Decide

Drift outside the 60/40 band? Payment due? It builds exactly one call — nothing else is possible.

03

Enforce

PolicyVault re-checks token, payee, cap and slippage on-chain, then reverts or settles atomically.

The interesting part is what is absent: no admin key in the loop, no off-chain allowlist to trust, no "please don't do that" in a system prompt.

05 — Live proof
Nanopayments

The agent pays
for its own data.

Steward doesn't get its FX rate for free. Each quote is bought from an x402-priced oracle at 0.0005 USDC per request, settled through Circle Gateway — machine to machine, no invoice, no subscription, no human.

Metered, not subscribed

An agent that runs 96 times a day shouldn't buy a monthly data plan. It should pay per decision — which is what a stablecoin L1 makes possible.

Degrades honestly

If the paid oracle is unreachable the agent falls back to a free source and says so — the dashboard badges the rate's provenance and totals what it spent.

06 — Agent Nanopayments
Why Arc

Gas in USDC changes
what an agent can be.

A treasury agent that must hold a volatile gas token to move stablecoins has a second treasury problem it never asked for. On Arc it holds one asset class and spends it — including on gas.
